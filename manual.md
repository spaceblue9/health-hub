# 📚 คู่มือการติดตั้งและใช้งาน Health Hub (สำหรับผู้เริ่มต้น)

ยินดีต้อนรับสู่โปรเจกต์ **Health Hub** ระบบจัดการข้อมูลสุขภาพครอบครัวและผู้สูงอายุ คู่มือนี้ออกแบบมาให้คุณทำตามได้ทีละขั้นตอน (Step-by-Step) แม้จะไม่มีความรู้เรื่องโค้ดดิ้งมาก่อนก็ตามครับ!

---

## 🛠️ ส่วนที่ 1: การเตรียมความพร้อม (สิ่งที่ต้องมี)
ก่อนเริ่ม โปรดตรวจสอบให้แน่ใจว่าคุณมีสิ่งเหล่านี้ในเครื่องคอมพิวเตอร์ของคุณ:
1. **Node.js** (โปรแกรมสำหรับรันเว็บแอป) - [ดาวน์โหลดที่นี่](https://nodejs.org/) (แนะนำเวอร์ชัน LTS)
2. **Git** (โปรแกรมจัดการเวอร์ชันโค้ด) - [ดาวน์โหลดที่นี่](https://git-scm.com/)
3. **โปรแกรม Code Editor** - แนะนำ [Visual Studio Code (VS Code)](https://code.visualstudio.com/)

---

## ☁️ ส่วนที่ 2: การตั้งค่า Supabase (ฐานข้อมูลของระบบ)
ระบบนี้ใช้ Supabase เป็นสมองหลักในการเก็บข้อมูล จัดการผู้ใช้ และระบบล็อกอินครับ

### 2.1 สร้างโปรเจกต์ใหม่
1. ไปที่เว็บ [Supabase.com](https://supabase.com/) แล้วสมัครสมาชิก / เข้าสู่ระบบ
2. กดปุ่ม **New Project** เลือกสถานที่ตั้ง (Region) เป็น **Singapore** (เพื่อให้โหลดเร็วที่สุดสำหรับคนไทย)
3. ตั้งรหัสผ่าน Database Password (จดรหัสผ่านนี้เก็บไว้ให้ดี)
4. รอประมาณ 2-3 นาทีจนกว่าโปรเจกต์จะสร้างเสร็จสมบูรณ์

### 2.2 คัดลอกกุญแจเชื่อมต่อ (API Keys)
1. ในหน้า Supabase ให้กดเมนู **Project Settings** (ไอคอนรูปฟันเฟืองด้านซ้ายล่าง)
2. เลือกแท็บ **API**
3. คัดลอกค่า 2 ตัวนี้เก็บไว้ใน Notepad ก่อนครับ:
   - `Project URL`
   - `Project API Keys (anon / public)`

### 2.3 สร้างตารางข้อมูล (รันคำสั่ง SQL)
เราต้องสร้างโครงสร้างฐานข้อมูล โดยให้ทำตามลำดับดังนี้:
1. กดเมนู **SQL Editor** (ไอคอนรูป `< / >` ที่เมนูด้านซ้าย)
2. กดปุ่ม **New Query**
3. ให้คุณเปิดไฟล์ในโฟลเดอร์ `database/` ของโปรเจกต์ แล้วก๊อปปี้โค้ดในไฟล์มารัน **ทีละไฟล์ตามลำดับ** ดังนี้:
   - `1.` รันไฟล์ `schema.sql` (สร้างตารางหลัก)
   - `2.` รันไฟล์ `schema-policies.sql` (สร้างกฎรักษาความปลอดภัย)
   - `3.` รันไฟล์ `update_permissions.sql` (เพิ่มฟังก์ชันแชร์สิทธิ์)
   - `4.` รันไฟล์ `co_caregiver_migration.sql` (เพิ่มระบบผู้ดูแลร่วม)
   - `5.` รันไฟล์ `add_manage_caregiver_permission.sql` (เพิ่มสวิตช์เปิดปิดสิทธิ์)
   - `6.` รันไฟล์ `add_delete_user_function.sql` (เพิ่มสิทธิ์ลบผู้ใช้งานให้ Admin)
   - `7.` รันไฟล์ `add_storage_admin_policy.sql` (เพิ่มสิทธิ์ให้ Admin จัดการ Storage ได้)

*(วิธีรัน: นำโค้ดมาวางในช่อง แล้วกดปุ่ม **Run** ที่มุมขวาล่าง พอมันขึ้น Success ค่อยลบโค้ดออกแล้วเอาไฟล์ถัดไปมาวางรันต่อครับ)*

### 2.4 สร้างถังเก็บรูปภาพและไฟล์แนบ (Storage)
1. ไปที่เมนู **Storage** (ไอคอนรูปกล่อง)
2. กดปุ่ม **New Bucket**
3. ตั้งชื่อถังว่า `medical_attachments` (สะกดให้ตรงเป๊ะนะครับ)
4. **สำคัญมาก:** ติ๊กเลือกช่อง **Public bucket** ให้เป็นสีเขียว แล้วกด Save

## 💻 ส่วนที่ 3: การเชื่อมต่อโค้ดเข้ากับ Supabase
ตอนนี้เรามีโค้ดอยู่ในเครื่องแล้ว และมี Supabase พร้อมแล้ว เราจะเอาสองอย่างนี้มาเชื่อมกันครับ

1. เปิดโฟลเดอร์โปรเจกต์นี้ด้วยโปรแกรม **VS Code**
2. เปิด Terminal ใน VS Code (เมนูด้านบน `Terminal` > `New Terminal`)
3. พิมพ์คำสั่ง `npm install` แล้วกด Enter (รอจนกว่าจะโหลดเสร็จ)
4. ให้สร้างไฟล์ใหม่ขึ้นมาในโฟลเดอร์นอกสุด ตั้งชื่อว่า `.env`
5. นำ API Keys ที่จดไว้จากข้อ 2.2 มาใส่ในไฟล์ `.env` ตามรูปแบบนี้:
   ```env
   VITE_SUPABASE_URL=ใส่_Project_URL_ตรงนี้
   VITE_SUPABASE_ANON_KEY=ใส่_anon_key_ตรงนี้
   ```
6. พิมพ์คำสั่ง `npm run dev` ใน Terminal แล้วกด Enter
7. ระบบจะแสดงลิงก์ `http://localhost:5173` ให้กด `Ctrl + คลิก` ที่ลิงก์เพื่อเปิดหน้าเว็บได้เลยครับ! 🎉

## 👑 ส่วนที่ 4: การตั้งค่า Admin คนแรก (สำคัญมาก!)
ระบบถูกออกแบบมาให้ **ความปลอดภัยสูงสุด** ดังนั้นคนที่สมัครเข้ามาใหม่ทุกคนจะอยู่ในสถานะ "รออนุมัติ (Pending)" ใช้งานอะไรไม่ได้เลย... รวมถึงตัวคุณเองที่เป็นคนสร้างเว็บด้วย! 

**วิธีแต่งตั้งตัวเองให้เป็น Admin:**
1. เปิดหน้าเว็บที่เรารันไว้ กดสมัครสมาชิกด้วยอีเมลของคุณให้เรียบร้อย (หน้าเว็บจะขึ้นว่ารออนุมัติ)
2. กลับไปที่เว็บ **Supabase**
3. เลือกเมนู **Table Editor** (ไอคอนตาราง) ด้านซ้ายมือ
4. คลิกเลือกตารางชื่อ `profiles`
5. คุณจะเห็นข้อมูลอีเมลที่คุณเพิ่งสมัครเข้ามา ให้ทำ 2 อย่างคือ:
   - ดับเบิ้ลคลิกช่อง **role** แล้วเปลี่ยนคำว่า `user` เป็น `admin`
   - ดับเบิ้ลคลิกช่อง **status** แล้วเปลี่ยนคำว่า `pending` เป็น `approved`
6. กลับมาที่หน้าเว็บของคุณ กดปุ่มรีเฟรช 1 ครั้ง... ยินดีด้วยครับ! ตอนนี้คุณมีสิทธิ์เข้าถึงเมนู **Admin Dashboard** แล้ว

---

## 👨‍👩‍👧‍👦 ส่วนที่ 5: วิธีการใช้งานของครอบครัว
เมื่อคุณตั้งค่า Admin คนแรกเสร็จแล้ว การใช้งานหลังจากนี้จะง่ายมากๆ ครับ ไม่ต้องกลับไปแก้ฐานข้อมูลอีกแล้ว

1. **เชิญคุณพ่อ/คุณแม่/ญาติ เข้ามาใช้ระบบ:**
   - ส่งลิงก์เว็บให้คุณพ่อ/แม่ 
   - ให้เขากดสมัครสมาชิกในหน้าเว็บ
   - หน้าจอเขาจะขึ้นว่ารออนุมัติ (ให้เขารอแป๊บนึง)
2. **การอนุมัติผู้ใช้ (ฝั่ง Admin):**
   - คุณกดเข้าเมนู **Admin Dashboard**
   - จะเห็นรายชื่อที่คุณพ่อ/แม่เพิ่งสมัครเข้ามา
   - คุณสามารถกด "เครื่องหมายถูกสีเขียว" เพื่ออนุมัติให้เขาเข้าใช้งานได้เลย
   - หากต้องการให้เขามีสิทธิ์ "แชร์ลิงก์" หรือ "จัดการผู้ดูแลร่วม" ก็ติ๊กถูกที่ช่องสิทธิ์ทางขวามือได้เลยครับ
3. **การสร้างโปรไฟล์ผู้ป่วย และการเชิญ "ผู้ดูแลร่วม":**
   - พ่อ/แม่ หรือคุณ สามารถสร้างโปรไฟล์ (เช่น สร้างชื่อคุณตา) ได้
   - หากต้องการให้ลูกหลานคนอื่นเข้ามาช่วยจดบันทึก ให้กดเข้าไปที่หน้าคุณตา > เลื่อนลงมากดปุ่ม **"จัดการผู้ดูแล"**
   - สร้างลิงก์คำเชิญ แล้วส่งผ่าน LINE ให้ลูกหลานคนนั้นกดรับสิทธิ์ได้เลยครับ

> [!TIP] 
> สำหรับผู้สูงอายุ ระบบของเรามีฟีเจอร์ **"ขยายขนาดหน้าจอ (Root Font Scaling)"** โดยสามารถกดเปลี่ยนขนาดตัวอักษรเป็น A+ ได้ที่เมนูแถบด้านซ้ายล่างครับ ระบบจะขยายทุกอย่างให้ตัวใหญ่เต็มตาและจำค่าไว้ให้ตลอดครับ!
